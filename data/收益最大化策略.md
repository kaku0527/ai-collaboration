## 📊 1. 填充率优化 (Fill Rate)

### 什么是填充率

**定义**: 成功展示广告的请求数占总请求数的百分比

**公式**: 填充率 = (展示数 / 请求总数) × 100%

**目标**: >90% (优秀), >80% (良好), <70% (需优化)

### 填充率优化策略

#### 策略1: 多广告尺寸组合

// 多尺寸请求示例 (Android)

AdView adView = new AdView(this);

#### 策略3: 请求时机优化

```
最佳请求时机:
✅ 应用启动时预加载
✅ 进入新页面时加载
✅ 广告展示前3-5秒加载
✅ 用户空闲时后台加载

❌ 避免时机:
❌ 用户正在操作时
❌ 网络状况差时
❌ 频繁重复请求
❌ 内存不足时
```

**代码示例**:
```java
// 预加载插页广告
public class AdPreloader {
    private InterstitialAd interstitialAd;

    public void preloadAd() {
        // 在应用启动或进入新场景时预加载
        AdRequest adRequest = new AdRequest.Builder().build();
        InterstitialAd.load(context, adUnitId, adRequest,
            new InterstitialAdLoadCallback() {
                @Override
                public void onAdLoaded(InterstitialAd ad) {
                    interstitialAd = ad;
                }
            });
    }

    public void showAdIfReady() {
        if (interstitialAd != null) {
            interstitialAd.show(activity);
        }
    }
}
```

## 💵 2. eCPM优化 (Effective CPM)

### eCPM影响因素

| 因素 | 影响 | 优化方法 |
|------|------|---------|
| **广告尺寸** | ⭐⭐⭐⭐⭐ | 选择高收益尺寸 |
| **广告类型** | ⭐⭐⭐⭐⭐ | 优先奖励视频 |
| **地理位置** | ⭐⭐⭐⭐⭐ | 定位到高价地区 |
| **用户价值** | ⭐⭐⭐⭐ | 用户分层定价 |
| **广告位位置** | ⭐⭐⭐ | 优化展示位置 |
| **时间因素** | ⭐⭐ | 利用高峰时段 |

#### 策略3: 动态底价 (Dynamic Floor)

```
动态底价设置:

什么是底价?
→ 广告接受的最低eCPM
→ 低于此价格不展示广告

设置原则:
├─ 初始设置: 基于历史数据
├─ 定期调整: 每周/每月
├─ 分地区设置: 不同地区不同底价
└─ 分广告位设置: 不同位置不同底价

底价调整策略:
1. 如果填充率>95%: 提高底价5-10%
2. 如果填充率<80%: 降低底价5-10%
3. 如果eCPM下降: 降低底价
4. 如果eCPM上升: 提高底价

示例:
地区: 美国
广告位: 游戏内插页
初始底价: $5
当前eCPM: $12
填充率: 92%
→ 建议底价: $7 (尝试提升)
```

#### 策略4: 竞价优化

```
统一拍卖 (Unified Auction):

概念:
→ 所有广告网络同时竞价
→ 出价最高者获胜
→ 最大化收益

配置步骤:
1. 在Ad Manager中启用统一拍卖
2. 添加所有广告网络
3. 设置各自的底价
4. 启用实时竞价

预期效果:
├─ eCPM提升: 20-30%
├─ 填充率提升: 10-15%
└─ 收益提升: 25-40%
```

## ⚖️ 4. 用户体验平衡

### 关键原则

```
用户体验与收益平衡:

核心原则:
✅ 用户留存 > 短期收益
✅ 长期价值 > 单次展示
✅ 用户满意度 > 广告数量

平衡策略:
1. 基于用户行为动态调整
2. 提供广告关闭选项
3. 奖励机制补偿用户
4. 尊重用户偏好
```

### 用户分层广告策略

```
用户价值分层:

高价值用户 (VIP/付费):
├─ 减少广告频率 (-50%)
├─ 提供免广告选项
├─ 优先保证体验
└─ 推广内购优惠

活跃用户 (非付费):
├─ 标准广告频率
├─ 奖励视频为主
├─ 插页广告适度
└─ 推广VIP优惠

新用户 (<7天):
├─ 减少广告频率 (-30%)
├─ 快速了解应用
├─ 提供新手福利
└─ 培养使用习惯

流失风险用户:
├─ 大幅减少广告 (-70%)
├─ 提供回归奖励
├─ 避免广告打扰
└─ 重新参与激励

实现代码示例:
```java
public class UserBasedAdController {
    public boolean shouldShowInterstitial(User user) {
        // 高价值用户
        if (user.isVip()) {
            return false;
        }

        // 新用户
        if (user.getDaysSinceInstall() < 7) {
            return Math.random() < 0.3; // 30%概率
        }

        // 流失风险用户
        if (user.getChurnRisk() > 0.7) {
            return false;
        }

        // 正常用户
        return true;
    }
}
```

### 广告疲劳度管理

```
疲劳度检测指标:
├─ 连续跳过广告次数
├─ 观看广告完成率
├─ 广告点击率
├─ 应用停留时间
└─ 用户评分变化

疲劳度应对策略:

轻度疲劳 (连续跳过2-3次):
├─ 降低广告频率 20%
├─ 增加奖励激励
└─ 尝试不同广告类型

中度疲劳 (连续跳过4-5次):
├─ 降低广告频率 40%
├─ 暂停插页广告
├─ 只展示横幅

重度疲劳 (连续跳过6+次):
├─ 停止展示广告 30分钟
├─ 提供免广告选项
├─ 重新评估策略

实现示例:
```java
public class AdFatigueManager {
    private int skipCount = 0;
    private long lastSkipTime = 0;

    public void recordSkip() {
        skipCount++;
        lastSkipTime = System.currentTimeMillis();
    }

    public boolean shouldReduceFrequency() {
        // 如果5秒内连续跳过3次
        if (skipCount >= 3 &&
            (System.currentTimeMillis() - lastSkipTime) < 5000) {
            return true;
        }
        return false;
    }

    public void reset() {
        skipCount = 0;
    }
}
```
