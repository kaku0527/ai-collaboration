## 一、核心概念定义

### 1.1 填充率 (Fill Rate)

**定义**: 成功展示广告的请求数占总请求数的百分比

**计算公式**:
```
填充率 = (广告展示数 / 广告请求数) × 100%
```

**行业标准参考**:

| 等级 | 填充率 | 评价 |
|------|--------|------|
| 优秀 | >95% | 国际一流水平 |
| 良好 | 85%-95% | 达标 |
| 一般 | 70%-85% | 需优化 |
| 较差 | <70% | 严重问题 |

**您的数据**: 填充率 24.79%（严重低于行业标准）

---

### 1.2 匹配率 (Match Rate)

**定义**: 广告请求被广告网络成功匹配并返回广告的比率

**计算公式**:
```
匹配率 = (返回广告的请求数 / 总请求数) × 100%
```

**行业标准参考**:

| 等级 | 匹配率 | 评价 |
|------|--------|------|
| 优秀 | >95% | 竞价充分 |
| 良好 | 85%-95% | 正常 |
| 一般 | 70%-85% | 需优化 |
| 较差 | <70% | 严重问题 |

**您的数据**: 匹配率 54.79%（严重低于行业标准）

---

### 1.3 关键指标关系

```
广告请求 → 匹配 → 广告返回 → 填充 → 展示
              ↓            ↓
           匹配率       填充率

总体效率 = 匹配率 × 填充率
```

## 二、低填充率/匹配率原因分析

### 2.1 常见原因

| 原因 | 影响程度 | 排查方法 |
|------|----------|----------|
| 广告位配置不当 | ⭐⭐⭐⭐⭐ | 检查广告单元设置 |
| 目标定位过窄 | ⭐⭐⭐⭐ | 放宽定向条件 |
| 底价设置过高 | ⭐⭐⭐⭐ | 逐步降低底价 |
| 广告网络不足 | ⭐⭐⭐ | 增加广告网络 |
| SDK版本过旧 | ⭐⭐⭐ | 更新至最新版本 |
| 请求时机问题 | ⭐⭐ | 优化预加载逻辑 |

### 2.2 开屏广告特殊说明

**您的数据**: 开屏广告eCPM $2.50，但填充率较低

**原因分析**:
1. **取值基数过大**: 开屏广告请求量大，但早期调优阶段配置不完善
2. **冷启动问题**: 新产品上线初期，竞价池数据不足
3. **竞品填充**: 初期流量质量不稳定，影响填充

**优化周期**: 通常需要2-4周调优期，观察趋势变化

## 三、趋势分析方法（必做）

### 3.1 趋势拆解原则

在分析填充率、匹配率时，**必须**进行以下拆解：

1. **按时间拆解**
   - 日趋势：观察短期变化
   - 周趋势：消除周末效应
   - 月趋势：整体趋势判断

2. **按产品拆解**
   - 各产品填充率对比
   - 各产品匹配率对比
   - 识别异常产品

3. **按地区拆解**
   - 高eCPM地区 vs 低eCPM地区
   - 填充率差异分析

4. **按广告类型拆解**
   - 开屏 vs 插页 vs 横幅
   - 视频 vs 图片

### 3.2 趋势判断标准

| 指标 | 上升趋势 | 下降趋势 | 稳定 |
|------|----------|----------|------|
| 填充率 | +5%/周 | -5%/周 | ±3% |
| 匹配率 | +3%/周 | -3%/周 | ±2% |
| eCPM | +10%/月 | -10%/月 | ±5% |
